卡片可能包含照片，文字和超链接。它们可能会显示包含大小可变的元素的内容，例如带有可变长度标题的照片。

卡片被拆分成了非常细的子组件。卡片的每个部分都是一个组件。可以按照示例的相同逻辑以任意顺序组合它们。

常规卡片

最简单的卡片由卡片头（header）、内容区（content area）和动作按钮（action buttons）构成，依次对应组件 md-card-header、md-card-content 和 md-card-actions。 卡片头可以包含标题和副标题，依次通过类 .md-title 和 .md-subhead 指定。内容区可以包含各种文本和图片。动作按钮通常包含 md-button 组件。

如果想在用户将鼠标悬停在卡片上时添加阴影（elevation），请添加 withHover属性。还可以添加波纹效果，只需要将 md-card 的包裹在 md-ripple 组件里。

带图片的卡片

为了给卡片增添视觉吸引力，可以在 md-header 标签内添加包含有图片（img 标签）的 md-card-media 组件。使用属性 md-medium 和 md-big 可以将图像大小增加到中等或大尺寸。

尽管对于纯文本标题不是必需的，但是当使用 md-card-media时，标题和副标题应包裹在 md-card-header-text 组件周围。

对于侧面带有操作图标按钮的无文本卡片，请将卡片内容包裹在 md-card-media-actions 组件内。

如果要使图像充满整个卡片区域，请将卡片内容包装在 md-card-media-cover 组件内。它的基本结构包括以下组件：

  • md-card-media，用于展示图片。图片的缩放比例可以通过 md-card-media 组件的 md-ratio 属性控制。
  • md-card-area，用于有卡片头（md-header）和动作（md-card-actions）的情况。

如果背景图片和标题之间的对比度不足，则后者将无法清晰辨认。为了解决这个问题，请使用 md-card-media-cover的属性 solid 或 textScrim。

不同底色的卡片

为卡片添加 .md-primary 或 .md-accent 类可以指定卡片底色。

可展开的卡片

为避免卡内信息过多或使卡片的高度不超过最大高度，可以将补充信息隐藏在可扩展区域内。

使用 md-card-expand 组件可以实现可展开卡片，同时需要搭配 md-card-expand-content 用于展示内容， 以及 md-card-action 用于提供一个触发展开动作的 md-card-expand-trigger 组件。
