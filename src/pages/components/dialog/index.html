对话框用于通知用户有关特定任务的信息，并且可能包含关键信息，需要做出决定或涉及多个任务。您可以在其中包含选项卡，所有表单组件等等。

对话框包含了一组模拟浏览器原生对话框的预设对话框，例如通知（alert）、确认（confirm）和提示输入（prompt）。

对话框

对话框组件内部可以包含任何 HTML 内容，用于创建丰富的对话框内容。

提示对话框

可以通过在 html 模板里使用 md-dialog-alert 组件， 也可以通过在 js 代码中直接调用 DialogAlert 组件的静态函数 show 来触发。

DialogAlert.show 函数第一个参数为必须的 Object 类型的 options， 该 options 可指定的属性和 md-dialog-alert 组件的属性一致。

该函数接收的第二个参数是可选的关闭对话框后的回调函数。 如果该回调函数返回了 false 则会阻止对话框的关闭。

确认对话框

展示确认对话框，可以通过在 html 模板里使用 md-dialog-confirm 组件， 也可以通过在 js 代码中直接调用 DialogConfirm 组件的静态函数 show 来触发。

DialogConfirm.show 函数第一个参数为必须的 Object 类型的 options， 该 options 里的可指定的属性和 md-dialog-confirm 组件的属性一致。

该函数接收的可选的第二个参数和第三个参数依次是确认和取消的回调， 如果回调函数返回了 false 则会阻止对话框的关闭。

在实际业务使用时，有一种常见情况是，在 confirm 回调中要调用服务器的 api 接口更新， api 请求成功后，才关闭对话框（如果失败，则允许用户重试）， api 请求过程中 Confirm 按钮不能点击且有 spinner 状态。

针对这种情况，confirmCallback 允许返回 false 来阻止对话框关闭，还允许直接 返回一个 Promise 对象。对话框会等待该 Promise，直到其 resolve 返回的数据 不是 false 才关闭对话框。

输入对话框

输入对话框和确认对话框类似，可以通过在 html 模板里使用 md-dialog-prompt 组件， 也可以通过在 js 代码中直接调用 DialogPrompt 组件的静态函数 show 来触发。

DialogPrompt.show 函数和 DialogConfirm.show 的参数一致，但有一个区别是： confirmCallback 如果返回字符串，会认为是对输入值的错误提示；如果返回 Promise，对于 reject 的数据或 resolve 的字符串 也会认为 是错误提示。
